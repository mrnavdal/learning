Lekce 02 · HTTP

HTTP server holýma rukama

Žádný Express. Nejdřív uvidíš, co server dělá na kost — pak oceníš, co za tebe framework dělá.

Most z Flutteru Roky jsi psal http.get('https://api…') — byl jsi klient, který se ptá. Teď jsi na druhé straně drátu: jsi server, ta věc, co odpovídá. Request/response je pořád ten samý pár, jen ho teď vyrábíš, ne konzumuješ.

Co server vlastně dělá?

Pod vší tou abstrakcí je server překvapivě jednoduchý. Dělá jen tohle, pořád dokola:

  1. Poslouchá na nějakém portu (např. 3000) — jako když si necháš otevřenou linku a čekáš na hovor.
  2. Když dorazí request, dostaneš dva objekty: req (co klient chce) a res (kam píšeš odpověď).
  3. Sestavíš response: status kód + hlavičky + tělo → a pošleš zpět.

Nejmenší možný server

V Node je vestavěný modul http — žádná instalace. Tady je celý „hello" server, přesně jak ho ukazuje oficiální dokumentace:

const http = require('node:http');

// Tahle funkce se zavolá při KAŽDÉM požadavku.
const server = http.createServer((req, res) => {
  res.writeHead(200, { 'Content-Type': 'text/plain' });
  res.end('hello');
});

server.listen(3000, () => {
  console.log('Poslouchám na http://localhost:3000');
});

To je vše. Devět řádků a máš běžící web server. Rozeberme tři klíčové kousky:

1 · req — co klient chce

Dvě věci z req budeš potřebovat pořád:

  • req.method → 'GET', 'POST', … (vzpomeň na kvíz z minula 🙂)
  • req.url → cesta, třeba '/users/42'

2 · res — odpověď, kterou vyrábíš

  • res.writeHead(200, { 'Content-Type': 'text/plain' }) → status kód + hlavičky naráz
  • (nebo jednotlivě: res.statusCode = 200 a res.setHeader('Content-Type', 'text/plain'))
  • res.end('hello') → pošle tělo a ukončí odpověď. Bez res.end() klient visí a čeká donekonečna.
Aha moment Odpověď je jen text, který sestavíš a pošleš: řádek se statusem, pár hlaviček, prázdný řádek, tělo. Žádná magie. Framework jako Express tenhle text jen skládá elegantněji.

Routing… ručně (a proč to bolí)

Reálný server nemá jednu odpověď pro všechno. Musíš se rozhodnout podle req.url a req.method. Bez frameworku to vypadá takhle:

const server = http.createServer((req, res) => {
  if (req.method === 'GET' && req.url === '/') {
    res.writeHead(200, { 'Content-Type': 'text/plain' });
    res.end('domovská stránka');
  } else if (req.method === 'GET' && req.url === '/users') {
    res.writeHead(200, { 'Content-Type': 'application/json' });
    res.end(JSON.stringify([{ id: 1, name: 'Ada' }]));
  } else {
    res.writeHead(404, { 'Content-Type': 'text/plain' });
    res.end('Not Found');   // 404 když nic nesedí
  }
});

Cítíš to? Pár cest a je z toho if/else peklo. Ruční parsování /users/42, čtení těla POST požadavku po kouskách, opakované hlavičky… Přesně tuhle bolest řeší Express — a proto ho v příští lekci oceníš. Ale teď už víš, co dělá pod pokličkou: nic magického, jen tohle if/else hezky zabalené.

🖐 Zkus si to (teď hned)

Tady je ta nejlepší zpětná vazba — spusť si to sám. V terminálu:

# 1. Vytvoř soubor
mkdir hello-server && cd hello-server
# vlož do server.js ten devítiřádkový příklad výše

# 2. Spusť ho
node server.js

# 3. V dalším terminálu (nebo v prohlížeči na localhost:3000):
curl http://localhost:3000
# → hello

Zkus změnit 'hello' na něco jiného, restartuj (Ctrl+C, znovu node server.js) a curlni znovu. Když tohle uvidíš na vlastní oči, „server" přestane být abstrakce.
